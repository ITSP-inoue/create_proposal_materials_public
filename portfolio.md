# 提案資料生成エージェント

― 生成AIを活用した提案業務効率化 PoC ―

---

## 1. ツール概要

本ツールは、自由記述で入力された文章をもとに、  
**一定フォーマットに沿った提案資料の下書きを自動生成**するエージェントです

---

## 2. 作成の背景

- マニュアル自動生成エージェントを以前のプロジェクトで利用
- 開発業務におけるドキュメントの生成を平準化するための基盤となる
- AIがブラックボックスにならず、例えばロゴや文言、CSS等手で修正可能な形

---

## 3. 導入前後の改善イメージ

### 導入前（Before）

- 資料作成が担当者のスキル・経験に依存
- 構成や表現のばらつきが発生
- 初稿作成に時間がかかる
- レビュー段階で「構成から修正」が発生しやすい

---

### 導入後（After）

- 自由記述からAIが構成と下書きを自動生成
- 提案資料の型が統一される
- 人は内容確認・調整に集中
- レビュー負荷・差戻し回数を低減

---

## 3. 費用面（試行利用想定）

### LLM利用コスト（試行フェーズ）

- 軽量モデル（例：gpt-4o-mini）を利用
- 数回の試行であれば **数円〜数十円程度**
- 実験用途では月額コストを極小化可能

### その他コスト

- ソフトウェア：オープンソース中心（MkDocs / Docker 等）
- インフラ：ローカル実行または既存環境で対応可能

> PoCフェーズでは **追加投資を抑えた検証**が可能

---

## 4. エージェント全体像

```

[自由記述入力]
↓
┌───────────────────┐
│ エージェント層 │
│ ① 構造抽出 │
│ ② 下書き生成 │
│ ③ 不足点指摘 │
└───────────────────┘
↓
[Markdown（docs/）]
↓
人によるレビュー・編集
↓
MkDocs build
↓
HTML / PDF 出力

```

### 設計上のポイント

- **正本は Markdown**（人がレビュー可能）
- AIは判断・決定を行わず、下書き支援に限定
- ビルド・出力は決定論的処理とし安全性を確保
- LLM有無を切り替え可能な構成
